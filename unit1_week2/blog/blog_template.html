PUTTING THE "CLASS" IN "CLASS"
APRIL 14, 2014

What are the best practices associated with using classes vs. ids?

-Dev Bootcamp Prompt

When I joined my first Guided Pairing Session with Greg and Ariel I relized I had only been using classes up to that point. I had never once used an id. This was technically working for me. None of my css files were too complicated so I could create a class for a single element without any negative consequences. However as Ariel explained to me, this isn't an ideal practice long-term. Ids are for single elements, whereas classes are for groups of them. They are defined slightly differently (ids have hash tags when being defined in CSS) but attributes are specified the same way. I've since found myself using ids a lot more, and I think my code is the better for it.
